Remotion — Video Hero Kaze Designs

Guía completa para crear el video hero de la homepage usando código React. Cómo, para qué y por qué.

🎯 ¿Qué es Remotion?

Remotion es un framework open-source que permite crear videos usando React (JavaScript). En lugar de editar video en Premiere o After Effects, escribes código que define cada frame, animación y transición.

React
Mismo lenguaje que el sitio Kaze
30fps
Renderiza frame por frame
MP4
Exporta directo a video

Piénsalo así: en vez de arrastrar capas en una línea de tiempo, escribes componentes React que se convierten en un video real.

La ventaja clave para Kaze

El video hero usa overlays glassmorphism — exactamente lo que CSS hace mejor que cualquier editor de video. Con Remotion, los paneles flotantes con blur son código nativo, no simulaciones.

🎬 ¿Para qué lo usaremos?

Para crear el video hero de 8-12 segundos que va en la homepage de Kaze Designs. El video tiene 5 actos:

Acto 1 — "La Prenda" (0s-2s)

Foto de producto/modelo + carrusel glassmorphism con prendas en blanco deslizándose desde la derecha.

Acto 2 — "El Color" (2s-4s)

Panel glassmorphism con círculos de color. Se selecciona uno → la prenda cambia de color (transición fade).

Acto 3 — "El Diseño" (4s-6s)

Panel vertical con opciones de logo/diseño. Se selecciona uno → aparece sobre la prenda.

Acto 4 — "El Neón" ⭐ (6s-9s)

El fondo se oscurece. Un rótulo neón se ENCIENDE con efecto glow (text-shadow animado). Brillo, reflejos, ambiente de impacto.

Acto 5 — "El CTA" (9s-12s)

Botón con gradiente animado (rosa→rojo→azul→verde). Texto "Cotiza tu Proyecto". Loop seamless.

Paso 3 — Configurar la composición

// src/Root.tsx
import { Composition } from "remotion";
import { KazeHero } from "./KazeHero";

export const RemotionRoot = () => {
  return (
    <Composition
      id="KazeHeroDesktop"
      component={KazeHero}
      durationInFrames={360}   // 12 seg × 30fps
      fps={30}
      width={1920}
      height={1080}
    />
  );
};

Paso 4 — Crear un acto (ejemplo: Acto 1)

// src/components/ActoPrenda.tsx
import { useCurrentFrame, interpolate, spring, useVideoConfig, Img, staticFile } from "remotion";

export const ActoPrenda = () => {
  const frame = useCurrentFrame();
  const { fps } = useVideoConfig();

  // Animación del carrusel: entra desde la derecha
  const slideIn = spring({
    frame,
    fps,
    config: { damping: 15, stiffness: 80 },
  });
  const carouselX = interpolate(slideIn, [0, 1], [600, 0]);

  // Opacidad del producto
  const opacity = interpolate(frame, [0, 15], [0, 1], {
    extrapolateRight: "clamp",
  });

  return (
    <div style={{
      width: "100%", height: "100%",
      background: "linear-gradient(135deg, #0a0a1a, #1a1a2e)",
      display: "flex", alignItems: "center", justifyContent: "center",
      position: "relative",
    }}>

      {/* Producto / Modelo */}
      <Img
        src={staticFile("modelo-camiseta.webp")}
        style={{ height: "80%", opacity, objectFit: "contain" }}
      />

      {/* Carrusel Glassmorphism */}
      <div style={{
        position: "absolute", right: 60, top: "50%",
        transform: `translateY(-50%) translateX(${carouselX}px)`,
        background: "rgba(255,255,255,0.12)",
        backdropFilter: "blur(20px)",
        border: "1px solid rgba(255,255,255,0.2)",
        borderRadius: 20, padding: "20px 30px",
        display: "flex", gap: 16,
      }}>
        {["camiseta","hoodie","gorra","polo","chaleco"].map((p) => (
          <div key={p} style={{
            width: 70, height: 90,
            background: "rgba(255,255,255,0.08)",
            borderRadius: 12,
            border: "1px solid rgba(255,255,255,0.15)",
          }} />
        ))}
      </div>
    </div>
  );
};

Paso 5 — Ensamblar los actos

// src/KazeHero.tsx
import { Sequence } from "remotion";
import { ActoPrenda } from "./components/ActoPrenda";
import { ActoColor } from "./components/ActoColor";
import { ActoNeon } from "./components/ActoNeon";
import { ActoCTA } from "./components/ActoCTA";

export const KazeHero = () => {
  return (
    <>
      <Sequence from={0} durationInFrames={75}>
        <ActoPrenda />
      </Sequence>

      <Sequence from={75} durationInFrames={60}>
        <ActoColor />
      </Sequence>

      <Sequence from={135} durationInFrames={60}>
        {/* ActoDiseno */}
      </Sequence>

      <Sequence from={195} durationInFrames={90}>
        <ActoNeon />
      </Sequence>

      <Sequence from={285} durationInFrames={75}>
        <ActoCTA />
      </Sequence>
    </>
  );
};

Paso 6 — Efecto neón (Acto 4)

// Efecto glow con CSS — el neón se "enciende"
const glowIntensity = interpolate(frame, [0, 30], [0, 1], {
  extrapolateRight: "clamp",
});

const neonStyle = {
  fontSize: 72,
  fontWeight: 800,
  color: `rgba(255, 62, 201, ${glowIntensity})`,
  textShadow: `
    0 0 ${10 * glowIntensity}px #ff3ec9,
    0 0 ${20 * glowIntensity}px #ff3ec9,
    0 0 ${40 * glowIntensity}px #ff3ec9,
    0 0 ${80 * glowIntensity}px rgba(255,62,201,0.4)
  `,
};

⚡ Tips para el equipo

⚠️ Errores comunes a evitar

  • No usar <img> de HTML — usar <Img> de Remotion para que espere a que cargue
  • No usar setTimeout ni setInterval — Remotion no funciona en tiempo real, usa frames
  • Las fotos deben estar en public/ y cargarse con staticFile("nombre.webp")
  • Para fuentes custom: usar @remotion/google-fonts o delayRender

✅ Mejores prácticas

  • Un componente por acto — mantener el código modular
  • Preview constante — npx remotion studio actualiza en vivo
  • Fotos en WebP — más ligeras, mejor rendimiento
  • Glassmorphism = CSS puro — backdrop-filter: blur() funciona perfecto en Remotion
  • Probar en 10fps primero — para iterar rápido, subir a 30fps para render final
