# Kaze Technical Edition

Use this skill to maintain the alternate Kaze design language: bright industrial, technical, editorial, and mobile-first.

## Visual Identity

- Position Kaze as a production system for custom apparel, signs, decals, DTF, engraving, and branded output.
- Use real shop imagery with overlay labels, pins, small specs, and process markers.
- Keep references generic to Kaze services. Remove visible machinery brand emphasis when possible in copy and layout.
- Use concise technical copy: title, one benefit sentence, optional spec tags.

## Palette

Prefer a brighter technical palette:

```css
--ink: #101114;
--panel: #181a1f;
--surface: #f4f1ec;
--signal-pink: #ff3f9a;
--electric-blue: #23b7ff;
--workwear-gold: #f4b63f;
--white: #ffffff;
```

## Overlay Pattern

Use images as explainers, not decoration:

- Add short labels like `THREAD DETAIL`, `MATERIAL CHECK`, `OUTPUT READY`.
- Use thin borders, small coordinates, numbered callouts, and compact chips.
- Make text large enough for mobile; avoid dense paragraphs over photos.
- Point overlays at process/value: color, finish, placement, material, durability, local production.

## Copy Pattern

```text
TECHNICAL TITLE
Short functional benefit that explains why the result matters.
```

Examples:

- `PRECISION STITCHING` / `Clean logo detail for uniforms, crews and brand gear.`
- `SIGN OUTPUT` / `Material-aware graphics for storefronts, vehicles and events.`
- `LOCAL PRODUCTION` / `A clear quote-to-finish flow from the Watsonville shop.`
