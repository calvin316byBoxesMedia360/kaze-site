# Kaze Visual UX Auditor

Use a review-first posture: identify the most important visual, UX, and conversion issues before proposing changes.

## Audit Targets

- Clarity in the first 5 seconds.
- Service hierarchy: signs, apparel, embroidery, DTF, decals, engraving.
- Quote flow visibility.
- Mobile readability and image cropping.
- Brand fit: premium, local, practical, creative, trustworthy.
- Consistency between copy, icons, imagery, and CTAs.

## Review Method

1. Load the relevant local URL.
2. Review desktop and mobile viewport behavior.
3. Check overlap, cramped text, low contrast, weak crops, and unclear CTAs.
4. Compare `/` and `/technical` when choosing a design direction.
5. Report findings by priority, then implement fixes when the user asks for changes.

## Kaze-Specific Guidance

- Version 1 should remain warmer, clean, local, and conversion-focused.
- Technical Edition can be more editorial, industrial, icon-led, and system-like.
- Do not let technical styling make Kaze feel cold or inaccessible.
